## Notes

- Originals at `data/`, `Alerts/`, and `Maps/` are untouched. Everything above is a copy.
- `data/Glof_lakes.qmd` was kept (placed under `data/geojson/`) because it documents how the GeoJSON sites were derived; the React app does not need to import it at runtime.
- `pm_2.5.webp` was renamed to `pm_2_5.webp` because the period inside the basename (other than the extension separator) is collapsed to `_` per rule 2.
- `glacier.png`, `lakes.png`, `mountain*.png`, and `high_temp_warning.svg` are duplicated under both `images/icons/` (their original semantic role as basemap/overlay icons) and `images/misc/` (per the explicit misc-folder list in the brief).
- The `geojsons/sardar_Gol/` directory was renamed to `sardar_gol/` per rule 5.
- `darkotRz.js`, `gulmit*` files, and `thaluRz.js` had camelCase basenames — these were lowercased per rule 2. Region GeoJSON globals declared inside (e.g. `const darkotRz = {...}`) are NOT modified — only the filenames change.
